# Prompt: Learn Stage - Improve the Test Automation Strategy

You are acting as a quality engineer in the final Learn stage of the
Holistic Testing Model.

During the previous stages, multiple testing and quality engineering
activities were executed, including:
- Exploratory testing
- Risk analysis
- T1&T5 test design
- Acceptance and regression testing
- Testing in production
- Log analysis
- Production validation

Your task is to analyze the collected findings and improve the existing
Test Automation Strategy.

---

# Part 1 - Reflect on the Findings

Review the previous testing outcomes and identify:
- Critical defects
- High-risk workflows
- Frequently failing functionality
- Regression-prone areas
- Production risks
- Stable and unstable features
- Business-critical user journeys

Analyze:
- What worked well
- What created risks
- Which areas require stronger automation support
- Which testing activities should remain manual or exploratory

---

# Part 2 - Improve the Test Automation Strategy

Adjust the Test Automation Strategy based on the findings.

Consider:
- Automation priorities
- Regression risks
- Business-critical workflows
- Production incidents
- Stability and maintainability
- Monitoring and feedback loops

Define:
- Which areas should be automated first
- Which risks require continuous monitoring
- Which tests provide the highest business value

---

# Part 3 - Define Top 10 Automation Candidates

Create a prioritized list of the 10 most important T1&T5 tests that
should be automated.

For each test provide:
- Test title
- Priority
- Short rationale
- Risk mitigated
- Business value protected

Focus on:
- High business impact
- High regression risk
- Frequently used workflows
- Production stability
- Security-critical functionality

---

# Important Instructions

- Work risk-oriented
- Base decisions on previous findings
- Prioritize business value
- Consider maintainability and long-term stability
- Use the T1&T5 naming syntax for test titles
- Be concise and structured

---

# Expected Output Structure

1. Reflection Summary
2. Updated Test Automation Strategy
3. Top 10 Automation Candidates
4. Key Learnings and Recommendations
