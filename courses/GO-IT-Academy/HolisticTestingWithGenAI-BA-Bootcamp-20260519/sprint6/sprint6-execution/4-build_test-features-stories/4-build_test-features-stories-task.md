# Hands-On: T1&T5 Test Design for Acceptance and Regression Testing

## Objective

Apply the T1&T5 test design technique to create a structured test set
for acceptance and regression testing of the actual sprint 6.

# Part 1 - Analyze the User Stories

Review the implemented user stories and identify:

- Which functionality requires acceptance testing
- Which existing functionality requires regression testing
- Which business processes are most critical
- Which risks should be prioritized

---

# Part 2 - Create T1&T5 Test Set

Select one dedicated use-case that is impacted by a user-story from Sprint 6 and create a structured
T1&T5 test set.

The test set shall contain:

- Acceptance test cases
- Regression test cases
- Risk classification
- Covered user stories
- T1&T5 title syntax

Use the following table format:

| Scope | Risk | Covered User Story | T1&T5 Title Syntax |
|---|---|---|---|
| Acceptance Testing | High | US-XXX | T1-Feature-Condition-ExpectedResult-T5 |
