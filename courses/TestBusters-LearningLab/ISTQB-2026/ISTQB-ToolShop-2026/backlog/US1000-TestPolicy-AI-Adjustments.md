# US1000: Extend Test Policy for Generative AI

**Epic:** AI Governance

## User Story

As a Head of Testing,

I want the Test Policy to include principles, responsibilities, and controls for using Generative AI in the software testing process ,
so that teams can use AI effectively, consistently, and responsibly.

---

## Acceptance Criteria

### AC1 - AI Fundamentals Covered

The Test Policy explains in glossary:

- Symbolic AI
- Classical Machine Learning
- Deep Learning
- Generative AI
- Large Language Models (LLMs)

### AC2 - LLM Types Defined

The Test Policy describes:

- Foundation Models
- Instruction-Tuned Models
- Reasoning Models

### AC3 - Multimodal AI Addressed

The Test Policy provides guidance for using:

- Multimodal LLMs
- Vision-Language Models

### AC4 - AI Usage in Testing Defined

The Test Policy identifies approved AI-supported activities such as:

- Test design
- Test case generation
- Test automation
- Test data generation
- Defect analysis
- Test reporting

and define the LLM's that is allowed to used.

### AC5 - Human Oversight Required

The Test Policy states that AI-generated outputs must be reviewed and approved by humans before use.

### AC6 - AI Chatbot Usage Governed

The Test Policy defines rules for using AI chatbots and LLM-based testing tools.

### AC7 - Data Protection Considered

The Test Policy includes guidance on handling confidential, sensitive, and personal data when using AI tools.

### AC8 - Traceability Required

The Test Policy requires documentation of significant AI-generated testing artifacts and decisions.

---

## Definition of Done

- Test Policy updated with Generative AI guidance.
- AI-related terminology standardized.
- Human oversight responsibilities defined.
- Data protection and traceability requirements included.
- Policy reviewed and approved by relevant stakeholders.
