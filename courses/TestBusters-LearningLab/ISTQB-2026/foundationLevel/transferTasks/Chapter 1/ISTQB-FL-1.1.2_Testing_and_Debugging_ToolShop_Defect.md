# Apply Testing and Debugging on Toolshop

## 1. Title
Apply Testing and Debugging on a Toolshop Defect

## 2. Syllabus Reference
ISTQB FL – 1.1.2 Testing and Debugging

## 3. Learning Objective
Apply ISTQB FL section 1.1.2 (Testing and Debugging) in a realistic webshop testing situation. Produce one concrete testing artifact that can be used by the Scrum team in the current sprint.

## 4. Context / Scenario
During the UAT of V5 some defects was found.

- Upload File in contact form
- In checkout invalid postal codes can be used
- After a password change in account, the user can not login anymore.
- In Account the password strenght feature shows wrong color.
- If the cart is not empty, and a new user register the new use get the cart from the previous user.

## 5. Task Instructions
- Pick 2 of the above defects
- Reproduce the defect and document it
- Debug the code to find the reason
- Document the reason 

## 6. Expected Outcome / Deliverable
One sprint-ready markdown artifact that documents the defect, the reason and the possible solution.
