# Create a System Prompt for GitHub Copilot with TTT426 Guardrails

## Title
Design a System Prompt for GitHub Copilot with Testing Guardrails

---

## Syllabus Reference
ISTQB GenAI –2.1.3

---

## Learning Objective
Understand how to control and guide AI-assisted development tools by
defining structured system prompts with testing-related guardrails.

---

## Context / Scenario
You are working in a team using GitHub Copilot to support test
engineering activities for the webshop
https://practicesoftwaretesting.com/.

The team observed that Copilot generates inconsistent test artifacts,
mixes test levels, and sometimes produces low-quality or risky outputs.

Your task is to design a **system prompt** that enforces basic
TTT426-related guardrails (Test Type, Test Technique, Test Level,
Test Data, Test Environment, Test Automation considerations).

---

## Task Instructions

1. Understand the goal:
   - The system prompt should guide Copilot when generating:
     - Test cases
     - Test ideas
     - Test automation code
   - The focus is on **quality, consistency, and safety**

2. Define the **core guardrails**:
   - Ensure the prompt enforces:
     - Clear test type (functional, non-functional, etc.)
     - Explicit test level (unit, integration, system, acceptance)
     - Usage of appropriate test techniques
     - Proper handling of test data
     - Awareness of test environment
     - Separation of manual vs automated tests

3. Create the **system prompt**:
   - Write a structured prompt that:
     - Defines the role of Copilot as a test engineering assistant
     - Includes rules and constraints (guardrails)
     - Specifies expected output formats
     - Prevents low-quality or unsafe outputs

4. Ensure the prompt includes:
   - Instructions to avoid assumptions
   - Requirement to make uncertainties explicit
   - Requirement for clear, structured outputs
   - Emphasis on test design before automation

5. Validate your prompt:
   - Review if your prompt would:
     - Improve consistency of outputs
     - Reduce ambiguity
     - Support junior testers

---

## Expected Outcome / Deliverable
A structured **System Prompt for GitHub Copilot** including:
- Role definition
- Clear guardrails based on TTT426
- Rules for output quality and structure
- Constraints to avoid poor or unsafe test artifacts


---

## Hints
- Think of the prompt as a **contract** between you and the AI
- Be explicit rather than generic
- Focus on guiding behavior, not just describing tasks
