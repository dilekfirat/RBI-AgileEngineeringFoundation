# Black Story Design Skills

## Skill: Create Black Story

### Goal

Create a fully structured Black Story.

### Steps

1. Define the hidden twist
2. Define the misleading assumption
3. Create a short teaser
4. Build the logical explanation
5. Add investigation clues
6. Define learning objectives
7. Rate difficulty

---

## Skill: Transform Real Incident Into Black Story

### Goal

Convert:
- production incidents
- testing failures
- security breaches
- agile anti-patterns

into a Black Story.

### Steps

1. Identify the surprising aspect
2. Hide critical context
3. Create curiosity
4. Preserve realism
5. Create investigative path

---

## Skill: Generate Workshop Warm-Up

### Goal

Create short Black Stories for:
- retrospectives
- training sessions
- testing workshops
- conference talks

### Constraints

- 5-10 minutes solving time
- high curiosity factor
- easy moderation

---

## Skill: Create Software Testing Black Story

### Focus Areas

- exploratory testing
- bug investigation
- production defects
- flaky tests
- CI/CD failures
- test data issues
- hidden workflows

### Expected Outcome

Participants should:
- ask investigative questions
- build hypotheses
- challenge assumptions
- think like testers

---

## Skill: Generate Educational Variants

### Variants

The same story can be generated as:
- beginner version
- expert version
- cybersecurity version
- agile version
- AI-risk version
- DevOps version

---

## Skill: Create Facilitation Notes

For each story generate:
- moderator hints
- common wrong assumptions
- escalation hints
- learning debrief questions

---

## Skill: Create Progressive Hint System

Hints should:
- gradually reveal information
- avoid spoilers
- support stuck participants
- preserve curiosity

Hint Levels:
- Level 1: tiny clue
- Level 2: directional clue
- Level 3: major insight
- Level 4: almost solved
